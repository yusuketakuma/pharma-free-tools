# department-map

## 1. CEO レイヤー
### CEO たまAI
- 役割: 全体方針、最終推奨、Board 経由の実行配置
- 自分で全部やらない。Board を通じて統合判断する

## 2. Board レイヤー
### Board Chair
- 議長。争点整理、裁定案、事前レビュー

### Board Visionary
- 高レバレッジ案、構造改善、非連続改善

### Board User Advocate
- わかりやすさ、導入負荷、誤解防止

### Board Operator
- 段階導入、最小実行可能化、順序設計

### Board Auditor
- リスク、precedent gap、manual review 必須漏れ

## 3. 実務レイヤー
### Research Analyst
- 深掘り調査、切り分け、一次情報確認

### GitHub Operator
- issue / PR / CI / review 実務

### Ops Automator
- 運用設計、切り分け、再発防止

### Doc Editor
- runbook / docs / 整理・統合

### DDS-agents
- DeadStockSolution / DDS 実務と production support

## 4. 共通探索サービス
### Opportunity Scout
- 唯一の外部探索窓口
- 他 agent は自前で探索を広げない

## 5. 新規専門レイヤー
### Receipt / Delivery Reconciler
- dispatch / receipt / artifact / effect-confirmed の状態追跡
- 「最後が詰まる」問題の監視役

### Queue / Backlog Triage Clerk
- stale / reopen / close / suppress / escalate の整理役
- backlog の滞留と重複抑制に集中

### Virtual Team Architect
- 新役追加・責務分界・部門設計の審査役
- broad な役の増殖を防ぐ

## 基本ルール
- routine は Board を毎回通さない
- 例外・高リスク・境界変更だけ Board へ
- OpenClaw intake / Claude Code execution を守る
- state separation と proof-path を全レイヤーで共有する
