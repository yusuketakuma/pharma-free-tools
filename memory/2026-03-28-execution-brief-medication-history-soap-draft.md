## Wireframe仕様（Board採用済み）
仕様詳細: `projects/pharma-free-tools/docs/pharmacy-medication-history-wireframe-proposal.md`

### 入力欄
1. **患者・処方の基本情報**: 年齢層、性別、既往/併存疾患、受診目的、処方変更の有無、直近の処方差分
2. **確認したい論点**: 副作用、効果実感、服薬状況、飲み忘れ/休薬、生活背景、残薬/期限切れ/受診間隔
3. **現場メモ**: 口頭で気になった点、医師に戻す点、患者へ説明したい点、次回までに確認したい点

### 出力欄
1. **SOAP下書き**: S(患者訴え/服薬状況/変化), O(処方差分/参考情報/観察), A(問題点要約/リスク判断/優先課題), P(次回確認/介入案/医師連絡要否)
2. **次回確認事項**: 次回来局時の確認ポイント、服薬継続確認、フォロー必要な副作用/症状
3. **患者説明メモ**: 1〜3文で使える説明文、平易な表現、「短く伝える版」と「丁寧版」の切り替え
4. **介入フラグ**: 医師確認必要/緊急度高/様子見可/追加情報待ち

### UI構成案
1. 上段: 患者・処方の入力欄
2. 中段左: 論点チェック
3. 中段右: SOAP下書き生成
4. 下段左: 次回確認事項
5. 下段右: 患者説明メモ
6. フッター: 保存用コピーボタン / リセット / 共有導線

## テンプレート文例（実装前準備）
SOAP生成ロジックには以下のテンプレート構造を組み込むこと。

### SOAP S テンプレート例
- 「[年齢層][性別]。[受診目的]にて来局。[服薬状況]。[気になる変化]。」
- 「前回処方から[変更内容]。[患者の訴え]。」

### SOAP A テンプレート例
- 「[疾患名]に対する[薬剤名]の治療において、[問題点]が懸念される。[優先課題]を優先して確認。」
- 「服薬アドヒアランスは[評価]。[リスク因子]があるため、[介入方針]とする。」

### 患者説明テンプレート例（短い版）
- 「今日のお薬は[変更点]です。[注意点]に気をつけてお飲みください。」
- 「[副作用症状]が出たら、すぐにご連絡ください。」

### 患者説明テンプレート例（丁寧版）
- 「今回の処方で[薬剤名]が[変更内容]になりました。[理由]のためです。[服用上の注意]についてご説明します。」
- 「[薬剤名]による[副作用]の可能性があります。[期間]は特に気をつけて、[対応方法]でご相談ください。」
