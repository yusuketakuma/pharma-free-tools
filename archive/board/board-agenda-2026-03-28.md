# 取締役会本会議議題: OpenClaw制御系とClaude Code実行系の運用体制

## 会議フロー

## 主要論点（最大3件）

### 1. **OpenClaw control plane vs Claude Code execution planeの明確な振り分け**

**基本方針:**
- 実行系エージェントは OpenClaw で指示を受け、Claude Code で実行する
- OpenClaw = control plane（制御系）
- Claude Code = execution plane（実行系）

**配置基準:**
| 役割 | 実行場所 | 判断基準 |
|------|----------|----------|
| repo調査・複数ファイル変更 | Claude Code | コード読解が必要 |
| テスト実行 | Claude Code | 実行環境が必要 |
| 実装・refactor | Claude Code | 多ファイル変更 |
| read_only/plan_only/short report | OpenClaw | 軽量作業 |
| lightweight coordination | OpenClaw | 短期連携のみ |

### 2. **差分指示要点の明示**

**指示の伝達方法:**
- Claude Code実行へ回す対象を具体的に示す
- 実行論点の配置判断理由を明確に記載
- 具体例:
  - 「backend API実装 → Claude Code実行」
  - 「ドキュメント整備 → OpenClaw完結」
  - 「テストスクリプト作成 → Claude Code実行」

**提案伝達ルール:**
- 自己改善proposalは直接適用せず、Board最終裁定の範囲だけを伝える
- 改善提案は「候補」として提示し、最終判断はBoardに委ねる

### 3. **成果物と実行状態の3段階管理**

**管理状態:**
| 段階 | 状態 | 内容 |
|------|------|------|
| 1段階 | 送信成功 | 指示の送信完了 |
| 2段階 | 受容成功 | 対象エージェントによる受領 |
| 3段階 | 成果物確認済み | 完了成果物の検収 |

**未完了追跡:**
- 未配信（送信未完了）
- 未受理（エージェント未受領）
- 未成果確認（完了未検収）

**自己改善proposal管理:**
- 提案内容の記録
- Board裁定时期の追跡
- 適用範囲の明示

---

## 最重要方針

### **実行系エージェントは OpenClaw で指示を受け、Claude Code で実行する**

**意味:**
- OpenClaw は統合管理・ルール制定・レビューを担当
- Claude Code は具体的なコード実行・テスト・実装を担当
- 誤った配置による効率低下を防ぐ

---

## アジェンダ別配置例

### Claude Code execution へ回す論点
- 新規APIエンドポイントの実装
- 既存コードのリファクタリング
- テストスクリプトの作成と実行
- 複数ファイルにわたる修正
- 性能改善の実装

### OpenClaw 完結でよい論点
- タスクの計画と分解
- ドキュメントの更新と整備
- ルールの制定とレビュー
- 仕様書の作成
- 軽量なcoordination

---

## 運用上の注意事項

1. **境界明確化**: 各タスクの配置判断理由を必ず記載
2. **効率最適化**: 不当な往復を減らし、一貫した運用を維持
3. **透明性**: 配置判断の根拠をドキュメントに明記
4. **継続改善**: 実績に基づいて配置ルールの見直しを実施
