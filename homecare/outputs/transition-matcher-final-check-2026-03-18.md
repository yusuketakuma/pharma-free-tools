## ゆうすけ操作手順（3ステップ）

### Step 1: MCSから処方データをエクスポート（約30分）

```
MedicalCareStation を開く
→ 患者検索 → 薬剤検索タブ
→ 検索条件: 処方中（終了日なし）の全患者
→ CSV形式でエクスポート
→ ファイル名: prescriptions_export.csv
→ 保存先: ~/Desktop/openclaw-cowork/homecare/
```

### Step 2: ファイル配置を確認（1分）

以下の3ファイルが同じフォルダにあることを確認:

```
~/Desktop/openclaw-cowork/homecare/
  ├── prescriptions_export.csv      ← Step 1で保存したもの
  ├── drugs_387_list.csv             ← 387品目リスト（※未完成の場合は薬剤師エージェントに伝達）
  ├── drugs_54_component_delete.csv  ← 54品目リスト（既存）
  └── transition-drugs-387-matcher.py ← 照合スクリプト
```

> ⚠️ `drugs_387_list.csv` がまだ完全版でない場合: 在宅患者で影響が大きい6成分（テリパラチド・アモキサピン・リラグルチド・ガランタミン・メマンチン・トラニラスト）は `drugs_54_component_delete.csv` に含まれているため、まずはこの状態で実行可能。

### Step 3: スクリプト実行（5分）

ターミナル（またはコマンドプロンプト）を開いて以下を実行:

```bash
cd ~/Desktop/openclaw-cowork/homecare/
python3 transition-drugs-387-matcher.py
```

実行完了すると、画面にサマリーが表示され、`transition_drugs_match_result.json` が生成される。

## 出力結果の読み方

スクリプト実行後、画面に以下の情報が表示される:

| 表示項目 | 意味 |
|---------|------|
| 総処方件数 | MCSからエクスポートした全処方の件数 |
| 387品目にマッチ | 経過措置対象の薬に該当した処方件数 |
| 影響を受ける患者数 | 対象薬を処方中の患者の人数 |
| **成分単位削除対象患者** | **最優先対応が必要な患者数**（名前と薬品名も表示） |

### カテゴリA〜D判定の意味

| カテゴリ | 患者への影響 | 次のアクション |
|---------|------------|-------------|
| **A: 成分単位削除** | 代替薬への切り替え必須 | 主治医へFAX（処方変更依頼）＋患者説明 → **3月末〜4月中** |
| **B: 早期切り替え推奨** | 高齢・認知症・在宅継続の患者 | 次回訪問時に代替薬提案 |
| **C: 経過観察** | 現状維持可（2027年3月末まで猶予） | 定期フォローで経過観察 |
| **D: 対応不要** | 既に切り替え済み or 処方終了 | 記録のみ |

**カテゴリAの患者がいた場合**: `teriparatide-action-kit-2026-03.md` のFAXテンプレート・患者説明テンプレートを使用してください。
