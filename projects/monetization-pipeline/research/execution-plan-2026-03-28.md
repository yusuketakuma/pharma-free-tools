## OpenClawが自律でやること（ジョブ設計）

### ジョブ1: 案件スクリーナー（Day 1-）
- **頻度**: 1日3回（朝9時・昼13時・夕18時）
- **内容**: クラウドワークス・ランサーズの「自動化」「AI」「Python」関連新規案件を検索・スコアリング
- **出力**: 提案候補リスト（案件名・報酬・期限・推奨度）
- **条件**: 単価5,000円以上・納期3日以上の余裕がある案件

### ジョブ2: 提案文ジェネレーター（Day 1-）
- **トリガー**: ジョブ1で提案候補が見つかった場合
- **内容**: 案件内容に基づくカスタム提案文を生成
- **テンプレート**: 課題理解 → 提案アプローチ → 工数・納期 → 強みの説明
- **出力**: 5件分の提案文（ゆうすけ確認待ち）

### ジョブ3: アフィリエイト商品リサーチャー（Day 1-2）
- **内容**: pharma-free-tools の各ツールページに関連する楽天市場商品を特定
- **出力**: 商品候補リスト（商品名・価格・リンク先・推奨挿入ページ）

### ジョブ4: HTML修正エンジニア（Day 2-3）
- **内容**: アフィリエイトリンク・広告枠を組み込んだHTML修正案を作成
- **出力**: 差分ファイルまたは修正後HTML（ゆうすけ確認待ち）

### ジョブ5: コンテンツライター（Day 3-6）
- **内容**: note販売用デジタルコンテンツの原稿執筆
- **品質**: AI検出回避（体験談・独自視点の追加）
- **出力**: Markdown原稿（ゆうすけレビュー待ち）

## リスクと制約

### 全般リスク
1. **時間不足**: ゆうすけの本業の負荷により作業が遅延する可能性 → 最小セットアップを3時間台に抑えた
2. **品質担保**: AI生成コンテンツの品質ムラ → 必ずゆうすけのレビュー工程を挟む
3. **収益規模**: 初期は数千円レベルの可能性 → 複数パスの並行実行でリスク分散

### パスA（クラウドソーシング）のリスク
- 実績ゼロでの受注はハードルが高い → 初期は投資価格（低価格）でレビューを回収
- スコープ肥大 → 事前に要件定義テンプレートを用意

### パスB（アフィリエイト）のリスク
- pharma-free-toolsの「pharma」がAdSense審査で不利になる可能性 → 楽天アフィリエイト（審査なし）で先行
- トラフィックが少ないと収益にならない → SEO改善は中期課題

### パスD（note）のリスク
- 集客が課題 → ココナラ・クラウドワークスのプロフィールから誘導
- 価格設定の難しさ → 500円〜1,500円の低価格帯でテスト

### 制約の確認
- ✅ 日本市場を基本
- ✅ 違法・グレー手法は除外
- ✅ 初期投資0円（全パス）
- ✅ OpenClaw + Claude Code の現行能力の範囲内
- ✅ 既存サイトを活用
