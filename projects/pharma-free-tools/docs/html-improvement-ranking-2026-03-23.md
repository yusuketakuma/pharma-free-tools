## 結論
既存74本のHTMLを再点検した結果、今回もっとも費用対効果が高いのは、**診断で止まっている高需要テーマを、実務でそのまま使える出力型ツールへ寄せる改善**です。

今回の上位5件は以下。

1. 薬歴下書き・要点整理支援
2. 返戻再請求ナビ
3. 在庫優先度ボード
4. 供給障害患者対応ワークベンチ
5. 服薬フォローアップ記録支援

評価軸は次の6点で統一した。
- UI/UXの古さ
- SEOの弱さ
- OGP / メタの弱さ
- CTA / 導線の弱さ
- テーマは良いのに完成度が低いか
- 実務インパクト（頻度・痛み・再訪可能性）

## 改善候補ランキング上位5件

### 1) 薬歴下書き・要点整理支援
- 対象HTML:
  - `pharmacy-medication-history-efficiency.html`
  - `medication-history-time-saving-checklist.html`
- 問題点:
  - 現状は「診断」中心で、SOAP下書き・患者説明メモ・次回確認事項の出力まで届いていない
  - 検索意図は強いが、ページ体験が「答えを得る」より「自己評価する」に寄っている
  - CTAが実務フロー直結ではなく、改善インパクトを取りこぼしている
  - OGP/Twitterメタは補強余地があった
- 期待改善効果:
  - 検索流入後にその場で使えるため、離脱率改善と再訪増が見込める
  - 「薬歴 時短」「薬歴 下書き」周辺の高意図流入と相性が良い
  - 診断ツールから実用ツールへの格上げ効果が最も大きい
- 修正コスト感: 中
- 優先度: 最優先

### 2) 返戻再請求ナビ
- 対象HTML:
  - `pharmacy-rejection-template.html`
  - `pharmacy-claim-denial-risk-diagnosis.html`
- 問題点:
  - テンプレートと診断が分散し、理由別ナビとして一気通貫になっていない
  - `pharmacy-rejection-template.html` に「返戣」表記ゆれが残っており、SEO/信頼感を損ねていた
  - canonical / OGP / Twitterメタが不足していた
  - 高意図テーマなのに、次に何をすべきかの導線が弱い
- 期待改善効果:
  - 「返戻」「再請求」など意図の強い検索に対してCVしやすい
  - 実務でのコピペ需要が高く、再訪利用も見込める
  - typo修正だけでも検索・信頼面のマイナスを回収できる
- 修正コスト感: 小〜中
- 優先度: 高

### 3) 在庫優先度ボード
- 対象HTML:
  - `pharmacy-inventory-diagnosis.html`
  - `pharmacy-reorder-point-calculator.html`
  - `inventory-order-optimization-checklist.html`
- 問題点:
  - 診断・計算・チェックが分かれており、最終的な「今日どれを優先対応するか」が見えない
  - `pharmacy-inventory-diagnosis.html` はSEO/OGPが弱く、診断後の行動CTAも抽象的
  - 在庫テーマは継続利用向きなのに、単発診断で終わりやすい
- 期待改善効果:
  - 発注・期限・欠品・高薬価を1画面で優先順位化できれば、実利用価値が高い
  - 継続利用・ブックマーク率が上がりやすい
  - 既存3本を束ねるだけでも完成度が大きく上がる
- 修正コスト感: 中
- 優先度: 高

### 4) 供給障害患者対応ワークベンチ
- 対象HTML:
  - `supply-disruption-patient-impact.html`
- 問題点:
  - テーマは非常に強いが、現状は「患者影響診断」で止まっている
  - 患者説明文・医師連絡文・薬歴記録文など、現場で欲しい出力がない
  - canonical / OGP / Twitterメタが不足していた
  - CTAが汎用的で、強い痛みに対して導線の押し込みが弱い
- 期待改善効果:
  - 差別化しやすく、SNS共有・自然被リンクも狙いやすい
  - 現場の説明・照会・記録をまとめて支援できれば既存診断群より価値が高い
  - supply shortage系の強い困りごとに対して、実行ツール化の伸びしろが大きい
- 修正コスト感: 中
- 優先度: 中高

### 5) 服薬フォローアップ記録支援
- 対象HTML:
  - `pharmacy-followup-efficiency.html`
  - `graceful-period-patient-followup-checklist.html`
- 問題点:
  - `pharmacy-followup-efficiency.html` は診断寄りで、記録文生成や次回確認項目の整理まで届いていない
  - OGPが汎用・英語寄りでテーマ一致度が弱かった
  - チェックリストはあるが、電話後の記録・共有導線が弱い
- 期待改善効果:
  - 義務業務の標準化ニーズに合い、再訪利用が期待できる
  - フォロー記録・未対応フラグ・次回確認事項まで出せると完成度が跳ねる
  - 管理薬剤師・対人業務強化文脈とも相性が良い
- 修正コスト感: 中
- 優先度: 中
