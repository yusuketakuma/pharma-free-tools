## 1. 服薬カレンダー生成ツール

### 1.1 リポジトリ

`pharma-homecare-calendar` → `https://yusuketakuma.github.io/pharma-homecare-calendar/`

### 1.2 目的

患者・介護者向けの服薬スケジュールを視覚的なカレンダー形式で生成し、印刷・共有できるツール。

### 1.3 機能一覧

1. **患者情報入力**（HomecareDBから読み込み可能）
2. **処方薬の服薬スケジュール設定**
   - 薬品名
   - 服用タイミング（朝食後/昼食後/夕食後/就寝前/食間/頓用）
   - 服用量
   - 服用日数
   - 特記事項（空腹時服用等）
3. **カレンダー生成**
   - 週間カレンダー（7日分、A4印刷対応）
   - 月間カレンダー（1ヶ月分）
   - 1日あたりの服用スケジュール表示
   - 服用済みチェックボックス（画面上）
4. **印刷機能**
   - A4縦 向き
   - 患者名・薬剤師名・生成日をヘッダーに表示
   - 大きな文字で高齢者にも見やすい
5. **エクスポート**
   - PNG画像として保存（html2canvas はフレームワーク不使用の制約に反するため、window.print で対応）
   - テキスト形式

## 2. 医師連絡文面生成ツール

### 2.1 リポジトリ

`pharma-homecare-contact` → `https://yusuketakuma.github.io/pharma-homecare-contact/`

### 2.2 目的

医師への連絡（疑義照会・副作用報告・処方提案・服薬状況報告）の文面をテンプレートから素早く生成するツール。

### 2.3 機能一覧

1. **連絡タイプ選択**
   - 疑義照会（用量・相互作用・重複など）
   - 副作用報告
   - 服薬アドヒアランス低下の報告
   - 残薬増加の報告
   - 処方変更提案
   - 退院時調整依頼
2. **患者情報入力**（HomecareDB連携）
3. **テンプレート選択**（連絡タイプごとに複数テンプレート）
4. **文面カスタマイズ**（テキストエリアで自由編集）
5. **出力**
   - コピー（電子カルテ貼り付け用）
   - テキストダウンロード
   - 印刷

### 2.4 テンプレート例

#### 疑義照会テンプレート
```
【疑義照会】
〇〇先生

患者：〇〇様（〇〇歳）
処方日：〇月〇日

以下の点につきまして、ご確認させていただけますでしょうか。

1. （具体的な疑問点）
   現在処方されている〇〇と〇〇の併用について、
   〇〇の相互作用が懸念されます。

2. （代替案の提案）
   〇〇への変更はいかがでしょうか。

お忙しいところ恐縮ですが、ご検討いただけますと幸いです。
よろしくお願い申し上げます。

〇〇薬局 薬剤師 〇〇
```

## 3. 在宅ケアチーム情報共有ツール

### 3.1 リポジトリ

`pharma-homecare-team` → `https://yusuketakuma.github.io/pharma-homecare-team/`

### 3.2 目的

訪問看護師、ケアマネージャー、理学療法士など在宅ケアチーム間での患者情報共有を支援するツール。
データのエクスポート/インポートで簡易的な情報共有を実現（サーバー不要）。

### 3.3 機能一覧

1. **チームメンバー管理**
   - 職種（薬剤師/看護師/ケアマネ/PT/OT/ST/医師/その他）
   - 名前
   - 所属
   - 連絡先
   - 担当患者との関連付け
2. **患者情報カード**
   - 基本情報（HomecareDB連携）
   - 主要な処方薬（3〜5剤に絞って表示）
   - 注意事項（アレルギー、腎機能等）
   - 直近の訪問記録サマリー
3. **共有情報のエクスポート/インポート**
   - JSON形式でエクスポート（メール/チャットで送付可能）
   - パスワード保護付きエクスポート（暗号化は平文でbase64エンコードのみ — セキュリティ要件は低い）
   - インポート時に差分表示
4. **引継ぎメモ**
   - チームメンバー間の引継ぎ事項
   - タイムスタンプ付き

## 4. 新規ツールの共通方針

- 全ツールで HomecareDB ユーティリティをインライン埋め込み
- フェーズ1の共通デザインシステムを適用
- SEO対応: meta description, OGタグ, 構造化データ
- ポータルサイトへの相互リンクを追加
- GAタグは実運用時に有効なIDを設定（開発中は未設定）
