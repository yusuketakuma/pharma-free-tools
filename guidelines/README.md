# guidelines

このディレクトリは、たまAIの仮想組織が共通参照する運用マニュアル群です。

重要: ここは外部記事のサンプル構成をそのまま写す場所ではありません。今の OpenClaw / Board / Claude Code / Telegram 運用に最適化した共通ルールを置く場所です。

目的は3つです。
1. 誰が担当しても判断のブレを減らす
2. Board / execution / report の意味を揃える
3. 新しい専用エージェントを追加しても、共通ルールを使い回せるようにする

## 使い方
- CEO / Board / 実務エージェントの全員が共通参照する
- 個別エージェントの役割定義より優先ではないが、共通ルールの正本とする
- 具体的な出力形式は `templates/` を参照する

## 優先順位
1. protected / manual review / trust boundary ルール
2. reporting / escalation / collaboration ルール
3. style / output quality ルール

## 自律改善ループ
- guidelines は仮想チームの本体として扱う
- wording / brand / reporting / research / escalation の改善は self-improvement 対象に含める
- step3 の内容は固定見本ではなく、今のシステムのボトルネックに合わせて継続改善する
- ただし protected path や統治ルール変更は manual review を優先する
