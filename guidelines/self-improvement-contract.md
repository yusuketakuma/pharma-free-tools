# self-improvement-contract

## 目的
self-improvement は agent の通常業務を置き換えるためではなく、運用の詰まり・ブレ・重複・誤読を減らすために行う。

## 改善対象
- guidelines の wording / reporting / brand / research / escalation ルール
- templates の required fields / structure / evidence path
- receipt / backlog / handoff / report の詰まり

## 改善対象にしないもの
- protected path の自動変更
- auth / approval / trust boundary / routing root の無承認変更
- Telegram 設定の自動変更

## 実務ルール
- 新しい問題が出たら、まず template で吸収できないかを見る
- 次に guideline で揃えられないかを見る
- それでも足りない時だけ agent 定義や新役追加を検討する
- guideline の変更提案は、何が良くなるか・何を防ぐか・影響範囲をセットで書く

## 完了条件
- wording を変えただけでなく、実際の report / handoff / verification に波及したことを確認する
- pending_artifact や manual_required の長期残留を減らす
