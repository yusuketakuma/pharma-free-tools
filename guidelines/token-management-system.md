# トークン管理システム

## 目的
- 継続稼働中のトークン消費の最適化
- 予測可能なトークン管理
- 自動的な稼働レベル調整

## 稼働モード

### 省エネモード
- **消費速度**: 8kトークン/時間 (33%削減)
- **サブエージェント**: 6体
- **実行間隔**: 7分
- **トリガー条件**: トークン残量 < 4時間分 or 月間予算 < 20%

### 通常モード (デフォルト)
- **消費速度**: 12kトークン/時間
- **サブエージェント**: 8体
- **実行間隔**: 5分
- **トリガー条件**: デフォルト

### 高効率モード
- **消費速度**: 16kトークン/時間 (33%増)
- **サブエージェント**: 10体
- **実行間隔**: 3分
- **トリガー条件**: トークン残量 > 12時間分 and 月間予算 > 80%

## アダプティブスケジューリング

### 基本ルール
1. トークン消費速度を5分ごとに測定
2. 予測モデルで残量を計算
3. モード切り替え条件を評価
4. サブエージェント間隔を動的調整
5. 状態変化時は通知

### 実行タイミング
- 測定: 毎時0分/15分/30分/45分 (15分ごと)
- 評価: 測定直後
- 調整: 即時反映
- 通知: 状態変化時のみ

## 予測バッファ管理

### バッファ設定
- **安全バッファ**: トークン残量の20%
- **警告ライン**: 4時間分の消費量
- **停止ライン**: 2時間分の消費量

### 予測モデル
```javascript
const predictConsumption = (currentRate, timeUntilRefill) => {
  const estimatedTotal = currentRate * (timeUntilRefill + 1);
  const buffer = tokensRemaining * 0.2; // 20%バッファ
  const safeHours = Math.floor((tokensRemaining - buffer - estimatedTotal) / currentRate);
  return { safeHours, riskLevel: estimatedTotal / tokensRemaining };
};
```

## 通知仕組み

### 通知タイミング
1. **モード変更時**: "トークン管理: [モード名]に切り替え"
2. **警告ライン**: "トークン残量警告: [残時間]分で警告ライン"
3. **停止ライン**: "トークン残量警告: [残時間]分で停止ライン"
4. **月予算警告**: "月予算警告: 使用率[使用率]%"

### 通知内容
- 現在モード
- トークン残量
- 予測残時間
- 次の補充予定
- 推奨アクション

## リスク管理

### 停止防止策
- バッファ計算で安全マージンを確保
- 予測モデルの精度継続改善
- 緊急時の手動オーバーライド機能

### 緊急時対応
1. **警告ライン到達**: 省エネモードへの自動切り替え
2. **停止ライン到達**: 全サブエージェントの停止
3. **月予算超過**: 即時停止と通知

### 手動オーバーライド
- CEOセッションから任意のモード設定可能
- 一時的な効率調整が可能
- 変更内容はログに記録

## 実装箇所

### 1. 監視レイヤー
- ファイル: `monitors/token-usage-monitor.js`
- 役割: トークン消費の15分ごとの測定

### 2. 実行レイヤー
- ファイル: `executors/token-mode-manager.js`
- 役割: サブエージェント数と間隔の動的調整

### 3. 通知レイヤー
- ファイル: `notifications/token-alerts.js`
- 役割: 状態変化時の自動通知

### 4. 予測レイヤー
- ファイル: `predictors/token-predictor.js`
- 役割: 消費予測とバッファ計算

## 連携エージェント

### 主要担当エージェント
- **ops-automator**: トークン管理システムの監視と実行
- **secretariat-hq**: 状態変化時の通知処理
- **supervisor-core**: 全体の監視と異常検知

### 連携タイミング
- 状態変化時: ops-automator → secretariat-hq (通知)
- 異常検知時: supervisor-core → ops-automator (調整指示)
- 手動操作時: CEO → 全エージェント (指示伝達)

## 自己改善ループ統合

### 定期レビュー
- **週次レビュー**: トークン管理の効率評価
- **月次レビュー**: 長期トレンドの分析
- **必要時に**: 即時的なパラメータ調整

### 改善提案
- 消費パターンの分析結果に基づくモード最適化
- 予測精度向上のためのモデル改善
- ユーザーフィードバックに基づいた調整

### 指標監視
- 平均消費速度
- モード切り替え頻度
- 予測精度
- 緊止動作の発生率

## トラブルシューティング

### 一般的な問題
1. **予測精度不足**: バッファ率を一時的に増加
2. **頻繁なモード切り替え**: モード切り替え閾値を調整
3. **通知過多**: 通知間隔を調整

### 緊急対応
1. **システム異常**: 手動モード設定へ移行
2. **予測不能な消費**: 一時的な省エネモード強制
3. **通知システム異常**: CEOへの直接通知
