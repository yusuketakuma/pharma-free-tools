# escalation-rules

## manual review 必須
- auth
- approval
- routing root
- trust boundary
- protected path
- Telegram 設定

## deep review 対象
- 本番影響が大きい
- rollback が難しい
- precedent 不足
- 複数層にまたがる変更

## low-risk assisted apply 対象
- docs
- runbook
- report wording
- template 追加
- state separation の明文化

## reopen 条件
- new evidence
- new owner
- due 変更
- success criteria 変更
- blocked dependency 発生
