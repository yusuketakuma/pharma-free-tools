# board-governance

## Board の役割
- routine の再審ではなく、例外・新規性・境界変更・高リスクを扱う
- dense communication / sparse deliberation を守る
- regular report は ledger-first の editor view とする

## Board に上げる条件
- precedent がない
- change scope が protected / root / trust boundary に触れる
- user-visible risk が高い
- contradiction / anomaly / drift がある

## Board に上げない条件
- 既存 precedent で流せる
- wording / report cleanup のみ
- low-risk docs / template 反映のみ

## 出力ルール
- approve / reject / revise は対象 proposal の要点を中身ベースで示す
- proposal_id, summary, proposed_changes, affected_paths, reason を持つ
