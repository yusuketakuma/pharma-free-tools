## 結論
今回の本会議アジェンダは、**主要論点を3件に圧縮**し、各論点について **OpenClaw 完結** か **Claude Code execution plane 行き** かを先に裁定して進める。

---

## 論点1: 実行系の配車・滞留・完了率
- 何を決めるか:
  - queue / rebalance / approval に滞留があるか
  - 再配車や優先順位変更が必要か
- 判定: **OpenClaw 完結**
- 理由:
  - control plane の裁定業務であり、repo 調査・実装・テストを伴わない
- 指示要点:
  - 滞留案件を洗い出し、止まっている理由を 1 行ずつ付ける
  - 再配車が必要なら、誰に何を振るかだけ決める

## 論点2: 直近重要課題の処置方針（最大3件まで）
- 何を決めるか:
  - いま本当に前に進めるべき案件を最大 3 件に圧縮する
  - 各件について「決定」「実行」「担当」「完了条件」を明確にする
- 判定: **案件ごとに分岐**
  - read_only / plan_only / short report / lightweight coordination → **OpenClaw 完結**
  - repo 調査 / 複数ファイル変更 / テスト / 実装 / refactor / heavy verification → **Claude Code execution plane**
- 理由:
  - 論点整理と裁定は OpenClaw が最速
  - 実装や重い検証は Claude Code execution plane に寄せる方が安全で再現性が高い
- 指示要点:
  - 各案件に必ず「OpenClaw 完結」または「Claude Code execution plane」を明記する
  - Claude Code 行きは repo / scope / expected output / test expectation をセットで渡す

## 論点3: 記録・差分指示・次回への固定化
- 何を決めるか:
  - 今回の裁定内容をどこへ記録するか
  - 次回再利用できる差分指示テンプレートを何にするか
- 判定: **OpenClaw 完結**
- 理由:
  - 記録・標準化・引き継ぎは control plane の責務
- 指示要点:
  - 本会議の結論を artifact / memory / proposal / runbook のいずれかへ残す
  - Claude Code へ回した案件は、投げた理由と期待成果物を必ず記録する

---

## Claude Code execution へ回す論点
- repo-wide 調査が必要な案件
- 複数ファイル変更が必要な案件
- テスト実行や検証が必要な案件
- 実装 / 修正 / refactor / heavy verification が必要な案件

## OpenClaw 完結でよい論点
- agenda seed の圧縮
- 優先順位づけ
- 進行役としての裁定
- short report / lightweight coordination
- 記録先決定と brief 化

## 実行面の配置判断理由
1. **OpenClaw は control plane**
   - 論点整理、優先順位、裁定、記録、差分指示作成を担う
2. **Claude Code は execution plane**
   - repo 調査、コード変更、テスト、実装、refactor を担う
3. **決定と実行を分離する方が事故が少ない**
   - OpenClaw が何をやるかを決め、Claude Code がどう実行するかを担う形が最も安定

## 本会議用の推奨裁定
- **最優先**: 論点2の対象案件を最大3件までに絞る
- **運用原則**: 実行を伴う案件は、OpenClaw で裁定後に Claude Code execution plane へ送る
- **即時アクション**: 今回は新規論点を増やさず、既存の滞留・重要案件の整理に集中する
