## 結論
- **優先順位は維持**。いま最も堅いのは **3. Claude Code・OpenClawセットアップ業務** と **4. クラウドワーク案件のOpenClaw自動化**。
- **Xの投稿自動化は引き続き最後尾**。実装は可能でも、X側の自動化規約が明確で、停止・審査・運用負荷のリスクが重い。
- **その他SNS自動化は「公式API前提なら可」だが、差分は小さい**。Meta系は professional/business account 前提、他媒体も審査/OAuth/権限制約が残る。
- **Claude Codeのセットアップ業務は商品化しやすくなっている**。公式ドキュメント上、terminal / IDE / desktop / web / Slack / GitHub Actions / GitLab CI/CD まで導線が広く、運用設計込みの支援にしやすい。
- **OpenClaw適合度** は、Claude Codeセットアップ > クラウドワーク自動化 > その他SNS > 薬局副業 > X の順で高い。

## 前回との差分
### 1) Claude Code・OpenClawセットアップ業務
- 公式ドキュメントで **利用面がより明確に広い** と確認できた。
- desktop app は **定期タスクのスケジュール** と **cloud sessions** を明示。
- web 版もあり、長時間タスクや並列実行の説明がある。
- 以前より「導入代行」だけでなく **運用設計・定例実行・CI連携** を含む提案がしやすい。

### 2) Xの投稿自動化
- X Developer Policy で、**write actions（投稿、DM、フォロー等）は Automation Rules 対象** と再確認。
- **明示的同意なしの自動返信/DM不可**、**bulk / aggressive / spammy 行為不可**、**複数アカウントでの同一・類似投稿不可** が明確。
- `POST /2/tweets` は継続提供され、**投稿作成/編集** と **paid partnership disclosure** には対応。
- つまり「できるが、事業としては運用コストが高い」状態は変わらず。

### 3) その他SNS自動化
- 前回の見立てどおり、**公式APIがある媒体は実装可能**。
- Instagram は **professional account 向け content publishing** が継続。
- ただし、**ビジネス/クリエイターアカウント、OAuth、審査、権限管理** がボトルネックのまま。
- 体感としては「規約が緩んだ」ではなく、**公式ルートを丁寧に通せば現実的** という整理。

### 4) クラウドワーク案件のOpenClaw自動化
- 大きな市場変化は未確認。
- ただし、公開情報収集・差分監視・定型レポート・表形式納品の相性は引き続き良い。
- 人手が必要なのは、本人確認、最終納品責任、クライアント独自ルール、秘密保持が絡む部分。

### 5) 薬局関連の副業リサーチ
- 大きな制度変更は未確認。
- 需要の中心は引き続き **薬局DX、FAQ、手順書、患者向け案内、制度整理、監修支援**。
- 個別患者判断や最終服薬指導は自動化の主対象にしない方針は維持。

## 優先順位の変化
### 変更なし
1. **Claude Code・OpenClawセットアップ業務**
2. **クラウドワーク案件のOpenClaw自動化**
3. **その他SNS自動化**
4. **薬局関連副業**
5. **Xの投稿自動化**

### 理由
- 1 と 2 は **収益化速度 / OpenClaw適合度 / 規約安定性** のバランスが最良。
- 3 は公式APIルートがあるが、審査・権限・素材確認が残る。
- 4 は高単価化の余地はあるが、案件化までの距離がやや長い。
- 5 は実装難易度より **運用リスク** が重い。

## 新しく見つかったリスク / 機会
### リスク
- **X**: 自動投稿そのものより、類似投稿・大量運用・自動DMが危険。
- **Claude Code**: 対応面が広いぶん、売り方を雑にすると「何でも屋」化して粗利が落ちる。
- **その他SNS**: 公式APIでも、アカウント種別や審査未通過で止まりやすい。
- **薬局**: 医療系は責任分界を曖昧にすると危ない。

### 機会
- **Claude Code + OpenClaw** を「導入」ではなく **導入 + 定時運用 + 通知集約 + 承認フロー** で売れる。
- **CloudWork系** は「公開情報収集」「比較表」「差分報告」で即PoC化しやすい。
- **SNS** は Instagram / LinkedIn などの **1媒体固定の承認付き予約投稿** が最小実験として現実的。
- **薬局** は監修そのものより、**制度整理と現場運用資料** に寄せると安全で売りやすい。

## 更新後の最小実験案
### 1. 最優先
**「90分で Claude Code + OpenClaw 導入」パッケージ**
- 内容: 初期設定、権限確認、基本運用、定例レポート1本、承認フロー1本
- 人手: ログイン、支払い、権限承認のみ
- 目的: すぐ売れる型を作る

### 2. 次点
**CloudWork向け 公開情報リサーチ自動化テンプレ**
- 入力: URL一覧 / 調査項目
- 出力: Markdown / TSV / 比較表
- 対象: データ収集、比較、分析、一覧化

### 3. SNS PoC
**Instagram か LinkedIn の 1媒体予約投稿**
- 投稿前承認つき
- 公式APIのみ
- まずは 1アカウント・1投稿種別に絞る

### 4. 薬局 PoC
**薬局向け FAQ / 手順書 / 制度整理の自動下書き**
- 電子処方箋 / オンライン服薬指導 / 患者向け案内
- 最終監修は人

### 5. X PoC
**低頻度・承認付きの投稿ドラフト運用**
- まずは自動投稿ではなく下書き生成まで
- 自動送信は後回し
