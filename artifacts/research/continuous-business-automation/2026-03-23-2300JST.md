## テーマ別メモ
### 1. Xの投稿自動化
- docs.x.com pricing で Free / Basic / Pro / Enterprise に加えて pay-per-use が明示。従量課金導線が増えた。
- docs.x.com create-post で投稿API自体は継続提供。
- 技術的には OpenClaw + cron + 承認フローで実装可。
- リスク: APIコスト変動、アカウント制限、スパム判定、UI自動化への依存禁止。
- OpenClaw単独完結度: 中（投稿生成〜予約〜記録は可、審査/炎上判断は人手必要）。

### 2. その他SNS自動化
- Instagram Content Publishing API はビジネスアカウント向け、画像/動画/リール/カルーセルに対応。
- TikTok Content Posting API は unaudited client では private 投稿のみ、audited で制限緩和。
- LinkedIn Share API は self-serve で `w_member_social` により投稿可。Rate limit 文書は 2025-08 更新。
- YouTube Data API upload も継続。
- OpenClaw単独完結度: 中〜高（公式APIがある媒体に限定すれば高い）。
- 人手必要: 初回OAuth、アプリ審査、素材品質確認、コメント返信、アカウント事故対応。

### 3. Claude Code・OpenClawセットアップ業務
- Anthropic docs では Claude Code が terminal / IDE / SDK / GitHub Actions / cloud sessions / scheduled tasks を含む広い導入対象になっている。
- 市場側も導入支援LP、AI導入相談、インストール代行サービスが見つかる。
- OpenClawを加えることで「導入」だけでなく「定時実行・通知集約・ブラウザ操作・運用設計」まで差別化可能。
- OpenClaw単独完結度: 高（提案、セットアップ手順化、運用runbook、定期ジョブ構築まで一気通貫）。
- 人手必要: ローカル権限承認、アカウントログイン、支払い/契約、最終運用責任の引き渡し。

### 4. クラウドワーク案件のうちOpenClawで完結しやすい領域
- CrowdWorks検索結果上でも データ検索/データ収集/リサーチ分析/データ入力/公開情報収集 系カテゴリが継続して厚い。
- これらは browser + extraction + validation + report で OpenClaw適合が高い。
- 特に「公開情報を拾う」「所定フォーマットに整形」「定期監視して差分報告」は OpenClaw向き。
- 低適合: 電話営業、本人確認前提、高精度画像判定、長文の完全手作業前提、秘密保持上ブラウザ持込不可案件。

### 5. 薬局関連の副業リサーチ
- CrowdWorks等で医薬品/EC商品監修案件が継続。
- 厚労省の電子処方箋ページは 2025-01 時点資料が見え、対応薬局リストは原則金曜更新。
- 厚労省のオンライン服薬指導情報は 2024-09 更新で継続整備。
- つまり薬局DX/オンライン運用/患者向け周知/教育資料の整備ニーズは続く。
- OpenClaw適合は「調査・規約整理・患者向けFAQ草案・薬局向け手順書・競合比較」で高い。
- 低適合は、個別患者への最終服薬指導や法的責任を伴う判断そのもの。
