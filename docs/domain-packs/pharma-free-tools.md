## Domain Overview

### 何を最適化する領域か
- 薬剤師の日常業務を支援するフリーツールの提供
- 業務効率化、コンプライアンス遵守、患者安全向上
- 業務フローに沿った機能と操作性の確保

### 主要 KPI / 成功条件
- ツール利用率: 80%以上
- ユーザー満足度: 4.2/5.0以上
- 業務効率化率: 60%以上
- エラー率: 0.5%以下

### やってはいけないこと
- 薬剤師の専門判断を置き換えること
- 現場の実情を無視した機能設計
- 法規制や倫理基準に違反する操作

## Known Failures

### 1. 用語の揺れ
**症状:**
- 同じ概念に対して複数の表現が存在
- 専門用語と日常用語の混在
- ユーザーが混乱する曖昧な表現

**原因候補:**
- チームメンバーの専門性の違い
- 文言一元管理の不備
- 機能追加時の一貫性チェック不足

**初動確認:**
```bash
# 用語辞書との照合
npm run terminology:check
# 重複表現の検出
npm run text:duplicates -- --path src/
```

**再現条件:**
- 新機能の追加時
- 複数チームメンバーの協業時
- ユーザーフィードバックに基づく修正時

**回避策:**
```bash
# 用語辞書に基づく修正
npm run terminology:fix
# 文言の一括置換
npm run text:replace -- --from "誤った表現" --to "正しい表現"
```

### 2. 現場フローとUI導線の不一致
**症状:**
- 実際の業務フローと操作手順が乖離
- 重要なステップがUIから見落とされる
- 余計なクリックが多い

**原因候補:**
- 開発者が現場の業務を理解不足
- UX設計の段階での現場確認不足
- アプリケーションと実務の概念の不一致

**初動確認:**
```bash
# UIフローの業務フローとの比較
npm run workflow:compare
# ユーザビリティテストの実行
npm run test:usability
```

**再現条件:**
- 新規機能のリリース時
- 顧客からのフィードバック時
- チームメンバーの入れ替え時

**回避策:**
```bash
# 業務フローの可視化
npm run workflow:visualize
# UI導線の再設計
npm run redesign:workflow -- --workflow-name [workflow-name]
```

### 3. 帳票出力の項目欠落
**症状:**
- 印刷用帳票に必要な情報が不足
- 項目配置が実務で使いにくい
- フォーマットが統一されていない

**原因候補:**
- 帳票要件の不十分な把握
- 印刷レイアウト設計の見落とし
- ユースケースの網羅性不足

**初動確認:**
```bash
# 帳票項目の網羅性チェック
npm run report:check-coverage
# 印刷プレビューの生成
npm run report:preview -- --report-name [report-name]
```

**再現条件:**
- 新規帳票の追加時
- 法規制変更時
- 顧客からの追加要望時

**回避策:**
```bash
# 帳票仕様の標準化
npm run report:standardize
# 印刷レイアウトの改善
npm run report:redesign -- --report-name [report-name]
```

### 4. 入力制約不足による運用事故
**症状:**
- 不適切なデータの入力許可
- 省略必須項目の未入力
- 数値範囲外の値の登録

**原因候補:**
- 入力検証ロジックの不備
- フロントエンドでのみのチェック
- バリデーションルールの一貫性不足

**初動確認:**
```bash
# 入力検証の網羅性チェック
npm run validation:completeness-check
# 不正データの挿入テスト
npm run test:validation -- --test invalid-data
```

**再現条件:**
- 新規フォームの追加時
- 既存フォームの変更時
- 外部システムからのデータインポート時

**回避策:**
```bash
# 入力制約の強化
npm run validation:strengthen -- --form [form-name]
# 入力チェックの追加
npm run validation:add -- --field [field-name] --rule [rule-name]
```

## Known Fix Patterns

### 1. 用語辞書に基づく修正
**典型修正:**
```bash
# 用語辞書の更新と適用
npm run terminology:update -- --source terminology-dictionary.csv
npm run terminology:apply
```

**修正時の副作用:**
- 既存コンテンツの一括更新
- ユーザーインターフェースの再レンダリング
- 翻訳ファイルの同期

**追加で見るべき関連箇所:**
- 用語辞書ファイル (`data/terminology/`)
- 国際化ファイル (`src/i18n/`)
- コンポーネントの文言 (`src/components/*/text.js`)

### 2. UI文言と業務フローを揃える修正
**典型修正:**
```bash
# 業務フローとUIの同期
npm run workflow:sync-ui
npm run ui:rephrase -- --workflow [workflow-name]
```

**修正時の副作用:**
- 操作手順の変更によるユーザー混乱
- ヘルプ文書の更新が必要
- トレーニング資料の修正

**追加で見るべき関連箇所:**
- ユーザーガイド (`docs/user-guide/`)
- ヘルプシステム (`src/components/Help/`)
- トレーニング動画 (`docs/training/`)

### 3. 入力チェック追加
**典型修正:**
```bash
# 入力検証ルールの追加
npm run validation:add -- --field dosage --rule required,min(1),max(1000)
npm run validation:add -- --field medication_name --rule required,exists-in-database
```

**修正時の副作用:**
- 既存データのバリデーション失敗
- ユーザー入力フォームの変更
- エラーメッセージの表示調整

**追加で見るべき関連箇所:**
- フォームコンポーネント (`src/components/forms/`)
- バリデーションライブラリ (`src/lib/validation/`)
- テストファイル (`tests/validation/`)

### 4. 手順説明/ヘルプの補強
**典型修正:**
```bash
# ヘルプコンテンツの追加
npm run help:add -- --topic [topic] --content [content]
npm run help:update -- --topic [topic] --version [version]
```

**修正時の副作用:**
- ヘルプシステムの表示更新
- ユーザーインターフェースの変更
- 検索インデックスの更新

**追加で見るべき関連箇所:**
- ヘルプコンポーネント (`src/components/Help/`)
- 検索機能 (`src/components/Search/`)
- ユーザーレビュー (`data/user-feedback/`)

## Comparison Axes

### 1. 現場の誤操作防止
**優先軸:**
- 誤操作防止: 入力制約、確認ダイアログ、デフォルト値の設定
- 操作効率: 入力補完、ショートカット、一括処理

**判断基準:**
- 安全性が重要な機能（薬品処方、数量入力）は「誤操作防止」を最優先
- 反復作業（データ入力、レポート生成）は「操作効率」を許容範囲内で調整

### 2. 学習コスト
**優先軸:**
- 直感的操作: 既存のUI/UXパターンの採用
- 効率的学習: チュートリアル、ヘルプ、ツールチップ

**判断基準:**
- 頻繁に使用する機能は直感的さを重視
- たまに使用する機能は学習支援を充実

### 3. 説明責任のしやすさ
**優先軸:**
- 透明性: 処理内容の可視化、理由の説明
- 簡潔性: 必要最小限の情報表示

**判断基準:**
- 医療現場では「透明性」を重視
- 緊急時には「簡潔性」を許容

### 4. 業務フローとの一致度
**優先軸:**
- 業務適合性: 現場のワークフローに沿った設計
- 機能豊富性: 多機能なアプリケーション

**判断基準:**
- 既存の業務プロセスを変えるリスクがある場合は「業務適合性」を優先
- 新規業務であれば「機能豊富性」も考慮

## Escalation Boundaries

### 自動で進めてよい範囲
#### 担当者: pharma-free-tools 専属担当
- UIの微調整（文言修正、レイアウト改善）
- 入力検証ルールの追加

- サンプルデータの更新
