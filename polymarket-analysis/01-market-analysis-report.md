# Polymarket BOT市場構造分析レポート

**作成日**: 2026-03-28
**対象**: Polymarket（グローバル版 / CTF Exchange）
**目的**: BOT取引による収益機会の評価と戦略検討

---

## 1. 市場構造概要

### 1.1 プラットフォーム基本情報
- **累積取引高**: $9B+（2024年時点）、月間アクティブトレーダー 314,500+
- **基盤**: Polygonチェーン上のCTF（Conditional Token Framework）Exchange
- **決済**: USDC（ステーブルコイン）
- **Resoltion**: UMA Oracle + Polymarket運営のハイブリッド

### 1.2 Negative Risk（Neg Risk）市場
複数結果イベント（選挙、スポーツなど）において、**一つの結果のみが勝つ**場合の資本効率化メカニズム。

- **No share → 他全結果のYes share変換**: ある結果のNoを1つ持つ = 他全結果のYesを1つずつ持つ、と等価
- **Augmented Neg Risk**: 事後的に新候補が出現する場合（選挙新立候補等）に対応。Placeholder + Explicit Otherで対応
- **価格制約**: 各結果のYes価格の合計は理論上$1に収束する必要がある（crossing ruleでMINT時は合計≥$1）
- **Merge/Split操作**: 合計価格が$1から乖離した際の裁定取引機会が存在

### 1.3 市場カテゴリ
| カテゴリ | 手数料 | 特徴 |
|----------|--------|------|
| 政治・地政学 | **無料** | 最大ボリューム、情報ベースのedgeが大きい |
| Crypto | Taker 0.25%（ピーク実効1.56%） | 高ボラティリティ、MM機会 |
| Sports | Taker 0.0175%（ピーク実効0.44%） | 15分短期市場、イベント駆動 |
| その他（科学・文化等） | 基本無料 | ニッチ、liquidityが薄い |

## 2. Fee Structure & リワード

### 2.1 現行手数料（2026年3月時点）
- **Geopolitical/World Events**: 手数料ゼロ
- **Crypto Markets**: Taker fee 0.25%（exponent=2, ピーク実効1.56%）
- **Sports Markets**: Taker fee 0.0175%（exponent=1, ピーク実効0.44%）
- **Maker**: 手数料ゼロ + Maker Rebate（Taker feeの20-25%を還元）
- **入出金**: Polymarket側は無料（Coinbase/MoonPay等の仲介手数料のみ）
- **Gas**: Polygonのため実質ゼロ

### 2.2 Maker Rebates Program
- Taker fee収入の一部を日次でMarket Makerに還元
- Crypto: Taker feeの20%、Sports: 25%
- 深い流動性とタイトスプレッドを提供するMMへのインセンティブ
- **注意**: 2026年3月30日に新fee構造が導入予定（変更の可能性あり）

### 2.3 BOT取引へのfee影響
- **Maker戦略**: 手数料ゼロ + rebate還元 = 非常に有利
- **Taker戦略**: 政治・地政学市場は無料、Crypto/Sportsはコストがかかる
- **結論**: 政治市場での情報ベース取引、または全市場でのMM戦略がfee面で有利

## 3. 戦略候補の切り分け

### 3.1 Market Making（MM）
| 項目 | 評価 |
|------|------|
| **概要** | bid-ask spreadで利益を獲得。Maker Rebateも加算 |
| **Edge源** | spread収入 + rebate + 時価変動によるinventory PnL |
| **必要資本** | 中〜大（流動性提供に十分な資金） |
| **リスク** | Inventory risk（一方向に偏った場合の評価損）、adverse selection |
| **複雑度** | 高（価格モデル、inventory管理、hedging） |
| **BOT要件** | 高頻度オーダー更新、低レイテンシ、状態管理 |
| **収益性** | spread × volume + rebate。政治市場はfee無料のため最も魅力的 |

### 3.2 Mispricing / Arbitrage
| 項目 | 評価 |
|------|------|
| **概要** | Neg Risk市場での価格乖離（合計≠$1）や、関連市場間の矛盾を利用 |
| **Edge源** | Neg Risk merge/split裁定、市場間矛盾 |
| **必要資本** | 中（同時ポジション建設に必要） |
| **リスク** | 取引コスト、実行リスク、ガス代（Polygonで実質ゼロ） |
| **複雑度** | 中（Neg Riskのcontract理解が必要） |
| **BOT要件** | 複数市場の価格監視、自動merge/split実行 |
| **収益性** | 小口だが低リスク。継続的モニタリングで安定収益の可能性 |

### 3.3 Information Edge（情報優位性）
| 項目 | 評価 |
|------|------|
| **概要** | 市場参加者より早い・正確な情報に基づく取引 |
| **Edge源** | ニュース解析、データ分析、ドメイン知識 |
| **必要資本** | 小〜中 |
| **リスク** | 情報の正確性、タイミング、市場の反応予測ミス |
| **複雑度** | 高（情報源の確保とリアルタイム解析が必要） |
| **BOT要件** | ニュースAPI連携、LLM解析、自動発注 |
| **収益性** | 大きいが継続的Edgeの維持が困難。AI/LLM活用で差別化の余地あり |

### 3.4 Event-Driven
| 項目 | 評価 |
|------|------|
| **概要** | 特定イベント（選挙結果、経済指標等）の前後に集中取引 |
| **Edge源** | イベント確率の過小/過大評価 |
| **必要資本** | 中 |
| **リスク** | イベント結果の不確実性、流動性急変 |
| **複雑度** | 中 |
| **BOT要件** | イベントカレンダー連携、条件付き発注 |
| **収益性** | 不定期だが大型イベントで高リターンの可能性 |

### 3.5 戦略推奨順位
1. **MM（政治・地政学市場）**: fee無料 + rebate、高ボリューム、BOT相性最良
2. **Mispricing/Arbitrage**: 低リスクで良い補助戦略。Neg Risk理解必須
3. **Information Edge（LLM活用）**: 差別化の鍵。初期投資必要
4. **Event-Driven**: 補助的に活用

## 4. 法規制・利用規約の精査

### 4.1 日本からの利用可能性
- **結論: 日本は利用可能地域に含まれる** ✅
- 2026年3月時点、日本はgeo-block対象外
- ただし、日本の賭博法・金融商品取引法との関係は**グレーゾーン**

### 4.2 日本法上のリスク評価
| 規制 | 影響 | 評価 |
|------|------|------|
| 賭博法 | 予測市場は「賭博」に該当する可能性 | ⚠️ グレーゾーン。過去に摘圧事例なし（海外事業者対象は不明） |
| 金融商品取引法 | 「デリバティブ取引」として規制される可能性 | ⚠️ CFTベースは整理対象外の可能性が高いが確証なし |
| 資金決済法 | 仮想通貨（USDC）の送受信 | リスク低（USDCは規制対象外の可能性が高い） |
| 税法 | 仮想通貨の雑所得/申告分離 | 確実に課税対象。適切に申告が必要 |

### 4.3 Polymarket ToS上の制約
- **VPN使用禁止**: ToS違反、アカウント停止のリスク
- **自動取引（BOT）**: ToS上で明示的に禁止されていない。APIが公開されている
- **市場操作禁止**: wash trading、spoofing等は禁止
- **KYC**: 現行グローバル版では不要（US版はKYC必要）

### 4.4 VPN使用に関する注意
- 日本は利用可能地域のためVPNは**不要**
- VPN使用はToS違反（規制地域からの回避目的に限定されるが、監視が強化される可能性）
- 日本国内IPでアクセスすれば問題なし

## 5. 競合状況と勝ち筋

### 5.1 予測市場競合
| プラットフォーム | 特徴 | BOT取引 | 競合度 |
|-----------------|------|---------|--------|
| **Polymarket** | 最大、オープンAPI、Neg Risk、fee無料市場多数 | ✅ API公開 | — |
| **Kalshi** | 米国CFTC認可、伝統的金融寄り | 限定 | 中 |
| **Metaculus** | コミュニティ予測、取引なし | N/A | 低 |
| **Azuro** | スポーツ特化、DeFiベース | 可能 | 低 |
| **PredictIt** | 学術目的、取引額制限 | ❌ | 低 |

### 5.2 既存BOTプレイヤーの推定
- MM系: 大手crypto MM firms（Wintermute, GSR等）が参入の可能性
- 個人BOT開発者: GitHubに複数のOSS Polymarket BOTが存在
- 機関投資家: CFTC認可後の米国参入で増加傾向

### 5.3 潜在的な勝ち筋
1. **ニッチ市場への特化**: 流動性が薄い市場でMM。大手がカバーしない領域
2. **LLMベースの情報解析**: ニュース・SNSのリアルタイム解析による情報edge
3. **Neg Risk裁定の自動化**: 複雑なNeg Risk構造の理解と実行を自動化
4. **日本時間の隙間**: 米国市場のオフ時間（日本時間昼）の流動性低下を狙う
5. **Cross-market analysis**: 関連市場間の矛盾を高速検出

## 7. 総合評価

### 推奨アプローチ
**段階的参入**を推奨：

1. **Phase 1（低リスク）**: Mispricing/Arbitrage BOTの開発・運用
   - Neg Risk市場のmerge/split裁定を自動化
   - 小資本で開始、リスク限定

2. **Phase 2（中リスク）**: 政治・地政学市場でのMM
   - fee無料 + rebateで有利な条件下でMM開始
   - inventory管理ロジックの実装

3. **Phase 3（差別化）**: LLM情報解析の統合
   - ニュース/SNSのリアルタイム解析
   - Information edgeの構築

### 成功の鍵
- Neg Riskメカニズムの深い理解
- 小さく始めてスケールするアプローチ
- リスク管理（ポジションサイズ、ストップロス）
- 日本法上の申告・税務対応の整備
