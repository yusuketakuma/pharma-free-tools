# 03. 勝ち筋エージェント実装計画
## ～薬局/訪問薬剤管理特化型～

**作成日**: 2026年3月6日
**担当**: AI社長

---

## 1. 目的

汎用AIではなく、薬局・訪問薬剤管理の現場で「確実に価値を出す」特化型エージェントを開発する。

### 解決する課題
- **現場で使われない**: 汎用すぎて実務に合わない
- **導入障壁**: 設定・学習コストが高い
- **ROI不明**: どれくらい効果があるか不明

### 「勝ち筋」の定義
```
勝ち筋 = 高頻度 × 高価値 × 低複雑性
```

---

## 2. ターゲット業務

### 2.1 優先順位付け

| 業務 | 頻度 | 価値 | 複雑性 | 優先度 |
|------|------|------|--------|--------|
| 服薬アドヒアランス確認 | 高 | 高 | 低 | **P0** |
| 薬剤相互作用チェック | 高 | 高 | 中 | **P0** |
| 訪問スケジュール最適化 | 中 | 高 | 中 | **P1** |
| 服薬指導レポート作成 | 高 | 中 | 低 | **P1** |
| 在庫管理・発注 | 中 | 中 | 中 | **P2** |

### 2.2 選定: 訪問薬剤管理支援エージェント

**理由**:
- 訪問薬剤管理は増加傾向
- 現場の負担が大きい
- AIの得意分野（データ統合・リスク検知）

---

## 3. エージェント設計

### 3.1 コア機能

#### 機能1: 服薬アドヒアランス分析
```
入力: 患者ID、服薬履歴
処理:
  - 処方通り服用しているか分析
  - 服薬漏れパターン検出
  - リスク要因の特定
出力: アドヒアランス率 + 改善提案
```

#### 機能2: リスク患者スクリーニング
```
入力: 患者リスト、医療データ
処理:
  - 多剤併用チェック
  - 薬剤相互作用スキャン
  - 高齢者特有リスク評価
出力: 優先訪問リスト
```

#### 機能3: 訪問レポート自動生成
```
入力: 訪問記録メモ（音声/テキスト）
処理:
  - 構造化データに変換
  - 重要事項の抽出
  - 次回アクション提案
出力: 訪問レポート + 次回計画
```

## 6. リスクと対策

### 6.1 技術リスク

| リスク | 確率 | 影響 | 対策 |
|--------|------|------|------|
| データ連携エラー | 中 | 高 | 段階的導入 + 手動確認 |
| 精度不足 | 中 | 高 | 100シナリオ評価 + 継続改善 |
| システム障害 | 低 | 高 | ロールバック機能 + 監査ログ |

### 6.2 運用リスク

| リスク | 確率 | 影響 | 対策 |
|--------|------|------|------|
| 現場の抵抗 | 高 | 中 | トレーニング + 段階導入 |
| 過度な依存 | 中 | 中 | 人間による最終確認 |
