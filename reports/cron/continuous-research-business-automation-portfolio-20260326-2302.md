# 継続リサーチ・ビジネス自動化ポートフォリオ — 2026-03-26 23:02 JST

## 結論
- いま一番価値が高いのは、**OpenClaw/Claude Code のセットアップ業務を標準化して外部化できる形にすること**。
- 収益化の近さでは、**CloudWork案件のうち OpenClaw で完結しやすい領域**（予約確認/追客/請求照合）が依然強い。
- **X投稿自動化** と **その他SNS自動化** は、広く取るほど規約/API依存で弱くなる。**公式API前提のチャネルだけ残す**のが正解。
- **薬局関連の副業** は、直接の臨床副業よりも、**知識商品・監修・業務支援ツール** の方が安全で再現性が高い。

## 前回との差分
### 1) Xの投稿自動化
- 変化は小さいが、**API取得/権限がボトルネック**である点がより明確になった。
- 先にやるべきは投稿ロジックではなく、**テンプレ整備と承認フロー**。
- OpenClaw適合度は高いが、**配信実行はAPI前提**でないと不安定。

### 2) その他SNS自動化
- 広くSNSを狙うより、**LINE公式アカウント / Instagram系の公式API / YouTube系の正規経路** に絞るべき。
- ブラウザ自動化やスクレイピング前提は、規約・保守・凍結リスクが高い。
- 「SNS自動化」単体の事業化より、**既存業務の配信補助** としての方が通りやすい。

### 3) Claude Code・OpenClawセットアップ業務
- ここは前回より前進。
- 重い investigate / fix / refactor を **Claude Code / acp_compat に寄せる方針** が明確化し、OpenClaw は control plane に寄せやすくなった。
- つまり、セットアップ業務は **手順書・handoff pack・verification pack の商品化** に向いている。

### 4) CloudWork案件のうちOpenClawで完結しやすい領域
- 3/23 baseline 以降も、上位は変わらず **予約確認/ノーショー対策、問い合わせ/見積追客、請求・入金消込**。
- 変化は「候補を増やす」より、**PoC接続に寄せるべき** という点が強まったこと。
- owner / due / success criteria がない案件は止まりやすい。

### 5) 薬局関連の副業リサーチ
- 既存の **pharma-free-tools** 資産があるので、ゼロからではない。
- ただし、今の勝ち筋は**臨床の直接副業** ではなく、**薬歴・監査・新人教育・在宅/服薬フォローを支える知識商品やツール**。
- 収益化速度は、プラットフォーム依存よりも**既存資産の小改修→公開→販売導線** の方が速い。

   - 手順を資産化できる

   - 早く売りやすい
3. **薬局関連の副業**
   - 既存資産があるので、知識商品化が現実的
4. **Xの投稿自動化**
   - 需要はあるがAPI/権限依存が重い
5. **その他SNS自動化**
   - 規約差が大きく、広く取るほど不安定

## 新しく見つかったリスク / 機会
### リスク
- X は API/権限取得が詰まると、実装が止まりやすい。
- SNS の広域自動化は、規約違反・凍結・保守負担で崩れやすい。
- 薬局系は、職場規定や法令解釈を雑に扱うと危険。

### 機会
- OpenClaw/Claude Code の組み合わせは、**setup-as-a-service** にしやすい。
- CloudWork では、**read-only PoC + 文面生成 + 台帳更新** の3点が再利用しやすい。
- 薬局系は、**実務知識をそのままコンテンツ・テンプレ・チェックリストに変換** できる。

## 更新後の最小実験案
### A. X投稿自動化
- 投稿テンプレを 10〜15本作る
- 公式APIの取得可否を先に確認
- 配信は「人手承認→自動実行」の2段階にする

### B. その他SNS自動化
- まず1チャネルだけ選ぶ
- 推奨は **LINE公式アカウント** か **Instagram系の公式経路**
- 1メッセージ種別だけでPoCする

### C. Claude Code・OpenClawセットアップ業務
- 1件を **handoff pack + verification pack + cleanup** で標準化
- 成果物は「手順書」「チェックリスト」「実行ログ」の3点に固定

### D. CloudWork自動化
- 20件の案件を Sheet で管理
- 予約/追客/請求のいずれか1領域だけで read-only PoC
- success criteria は「漏れゼロ」「下書き生成率」「人手修正率」で見る

### E. 薬局関連の副業
- 1テーマだけ選ぶ
- 推奨は **薬歴下書き / 監査チェック / 新人教育** のいずれか
- 既存HTMLやテンプレを 1枚の販売物に落とす
