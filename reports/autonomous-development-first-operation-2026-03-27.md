# 自律開発本部 初回実務

- 仕様→実装→検証の流れを1本通すための最小構成を整理した

- DDS担当
  - 実装・実務判断
- 変更検証担当
  - verification / smoke / effect 確認
- 文書担当
  - runbook / report 反映
- 組織設計担当
  - 役割や責務分界の見直し

## 最初に通すべき最小の変更種類

- proof-path / state separation / handoff の改善

## 検証担当の使い方
- apply 完了を success とみなさず、verification を別で確認する
- pending_artifact を長引かせない

## 次に行うこと
