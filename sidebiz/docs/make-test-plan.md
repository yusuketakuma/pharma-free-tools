# Make無料枠検証計画

## 目的
SNS自動投稿のPoC（概念実証）で、収益化自動導線の基盤構築

## Make無料枠の制限

- **オペレーション**: 1,000回/月
- **データ転送**: 100MB/月
- **実行時間**: 無制限
- **接続数**: 無制限

## 検証シナリオ

### Scenario 1: RSS → X（Twitter）自動投稿

**目的**: ブログ更新時に自動でXへ投稿

**フロー**:
1. RSS Trigger（1時間ごとポーリング）
2. Filter: 新規記事のみ
3. Text Parser: タイトル + URL + ハッシュタグ生成
4. X（Twitter）: 投稿

**想定オペレーション数**:
- 1回実行: 4オペレーション
- 1日24回: 96オペレーション
- 月間: 約2,880オペレーション → **無料枠超過**

**対策**:
- ポーリング間隔を3時間に延長（月間960オペレーション）
- または新規記事のみフィルタリングで実行頻度削減

### Scenario 2: Notion → X自動投稿

**目的**: Notionで下書きした投稿をXへ自動公開

**フロー**:
1. Notion Trigger（ステータス変更監視）
2. Filter: ステータス = "投稿待ち"
3. X（Twitter）: 投稿
4. Notion: ステータスを"投稿済み"に更新

**想定オペレーション数**:
- 1回実行: 3オペレーション
- 週3投稿: 9オペレーション/週
- 月間: 約36オペレーション → **無料枠内**

**判定**: Scenario 2 が無料枠検証に最適

## 検証手順

### Step 1: アカウント作成（ユーザー依存）
1. https://www.make.com でアカウント作成
2. 無料プラン選択

### Step 2: シナリオ2実装
1. Notion接続設定
2. X（Twitter）接続設定（Developer PortalでAPI Key取得必要）
3. フロー作成
4. テスト実行

### Step 3: 動作確認
1. Notionで「投稿待ち」ステータスのレコード作成
2. 自動投稿されるか確認
3. ステータスが「投稿済み」に変わるか確認
