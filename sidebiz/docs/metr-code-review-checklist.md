# METR研究に基づくAI生成コード レビューチェックリスト

## 背景

METR研究（Many SWE-bench-Passing PRs Would Not Be Merged into Main, 2026/3/10）により、SWE-bench合格の約半数がメンテナーによって却下されることが判明。自動採点スコアは実務有用性を過大評価している。

## 却下理由Top3

1. **コア機能不全** - 主要機能が正しく動作しない
2. **他コード破壊** - 既存機能への悪影響
3. **コード品質問題** - 可読性・保守性・テスト不足

## レビュー観点（3点チェックリスト）

### 1. コア機能確認

- [ ] 主要機能が仕様通りに動作するか
- [ ] エッジケースでクラッシュしないか
- [ ] ユーザーが期待する出力が得られるか

### 2. 他コード影響確認

- [ ] 既存機能への影響がないか
- [ ] 依存関係が破壊されていないか
- [ ] パフォーマンス劣化がないか

### 3. コード品質確認

- [ ] 可読性（命名・構造・コメント）
- [ ] テストカバレッジ（単体テスト・統合テスト）
- [ ] ドキュメント（README・APIドキュメント）

## sidebiz適用指針

### 副業担当（開発時）

1. **Quint 4ステップワークフロー**との組み合わせ
   - Step 2（仕様検証）でコア機能確認
   - Step 4（コード検証）で3点チェックリスト実施

2. **テスト必須**
   - 全ツールにlocalStorage保存テスト追加
   - クロスブラウザ動作確認（Chrome/Firefox/Safari）

### 保守担当（監査時）

1. **週次監査**
   - 新規ツールの3点チェックリスト確認
   - 既存ツールの回帰テスト

2. **品質ゲート**
   - コア機能不全: 即座修正
   - 他コード破壊: ロールバック検討
   - コード品質問題: 優先度付けで対応
