## 2026-03-06 22:40 JST

---

## 前提

- Notionアカウント作成済み（無料プランでOK）
- PCブラウザ推奨

---

## 手順1: 新規ページ作成

1. Notionを開く
2. 左サイドバーの「+」をクリック
3. ページ名を入力: **薬剤師業務管理システム**
4. アイコンを設定: 💊

---

## 手順2: データベース作成（5つ）

### 2-1: 患者カルテ

1. ページ内で `/table` と入力 → 「テーブル - フルページ」を選択
2. データベース名: **患者カルテ**
3. プロパティを追加:

| プロパティ名 | 型 | 設定 |
|-------------|---|------|
| 患者ID | タイトル | 自動設定 |
| 氏名 | テキスト | - |
| 生年月日 | 日付 | 日付形式 |
| 在宅フラグ | チェックボックス | - |
| 備考 | テキスト | - |
| 最終来局 | 日付 | 日付形式 |

4. `patients.csv` をインポート:
   - 右上の「...」→「インポート」→「CSV」
   - `patients.csv` を選択

### 2-2: 処方箋ログ

1. 同様に `/table` で新規データベース作成
2. データベース名: **処方箋ログ**
3. プロパティ:

| プロパティ名 | 型 | 設定 |
|-------------|---|------|
| 処方日 | タイトル | 日付形式 |
| 患者 | 関連 | 「患者カルテ」を選択 |
| 薬品リスト | マルチセレクト | - |
| 疑義照会 | チェックボックス | - |
| 薬歴 | テキスト | - |
| 担当薬剤師 | ユーザー | - |

4. `prescriptions.csv` をインポート

### 2-3: 在庫管理

1. `/table` で作成
2. データベース名: **在庫管理**
3. プロパティ:

| プロパティ名 | 型 | 設定 |
|-------------|---|------|
| 薬品名 | タイトル | - |
| 在庫数 | 数値 | - |
| 発注点 | 数値 | - |
| メーカー | セレクト | - |
| 更新日 | 日付 | - |

4. `inventory.csv` をインポート

### 2-4: 学習ログ

1. `/table` で作成
2. データベース名: **学習ログ**
3. プロパティ:

| プロパティ名 | 型 | 設定 |
|-------------|---|------|
| タイトル | タイトル | - |
| カテゴリ | セレクト | 疾患/薬効/法規/実務 |
| 日付 | 日付 | - |
| ソース | URL | - |
| メモ | テキスト | - |

4. `learning.csv` をインポート

### 2-5: タスク管理

1. `/table` で作成
2. データベース名: **タスク管理**
3. プロパティ:

| プロパティ名 | 型 | 設定 |
|-------------|---|------|
| タスク | タイトル | - |
| ステータス | セレクト | 未着手/進行中/完了 |
| 優先度 | セレクト | 高/中/低 |
| 期限 | 日付 | - |
| 関連患者 | 関連 | 「患者カルテ」を選択 |

4. `tasks.csv` をインポート

## 手順3: トップページ構成

メインページに以下を配置:

### 3-1: 本日のタスク

1. `/linked` と入力 → 「リンクされたデータベース」を選択
2. 「タスク管理」を選択
3. フィルタ設定:
   - ステータス ≠ 完了
   - 期限 = 今日

### 3-2: 在庫アラート

1. `/linked` → 「在庫管理」を選択
2. フィルタ設定:
   - 在庫数 < 発注点（※要計算プロパティ追加）

**計算プロパティ追加方法:**
1. 在庫管理データベースに「要発注」プロパティを追加（数式）
2. 数式: `prop("在庫数") < prop("発注点")`

### 3-3: 最近の処方箋

1. `/linked` → 「処方箋ログ」を選択
2. ソート: 処方日（降順）
3. 表示件数: 5件

### 3-4: 学習進捗

1. `/linked` → 「学習ログ」を選択
2. グループ化: カテゴリ
3. ソート: 日付（降順）

---

## 手順4: ビューのカスタマイズ

各データベースに複数ビューを作成:

### 患者カルテ
- **全件**: テーブルビュー
- **在宅患者のみ**: フィルタ（在宅フラグ = TRUE）
- **最近来局**: ソート（最終来局 降順）

### 処方箋ログ
- **全件**: テーブルビュー
- **疑義照会あり**: フィルタ（疑義照会 = TRUE）
- **患者別**: グループ化（患者）

### 在庫管理
- **全件**: テーブルビュー
- **要発注**: フィルタ（要発注 = TRUE）
- **メーカー別**: グループ化（メーカー）

### 学習ログ
- **全件**: テーブルビュー
- **カテゴリ別**: グループ化（カテゴリ）
- **ギャラリー**: カード形式で表示

### タスク管理
- **全件**: テーブルビュー
- **ボード**: ステータス別（かんばん形式）
- **優先度別**: グループ化（優先度）

---

## 手順5: テンプレートボタン設定

よく使う入力パターンをテンプレート化:

1. 各データベースの右上「↓」→「テンプレート」
2. 新規テンプレート作成:

### 患者カルテ用
- 名称: **新規患者（在宅）**
- デフォルト値:
  - 在宅フラグ: TRUE
  - 備考: アレルギー確認済み

### 処方箋ログ用
- 名称: **定期処方**
- デフォルト値:
  - 疑義照会: FALSE

### タスク管理用
- 名称: **在宅訪問タスク**
- デフォルト値:
  - 優先度: 高
  - ステータス: 未着手

## トラブルシューティング

### CSVインポートが失敗する
- 文字コード: UTF-8で保存し直す
- 日付形式: YYYY/MM/DDに統一

### 関連プロパティが表示されない
- 両方のデータベースが同じワークスペースにあるか確認

### フィルタが効かない
- プロパティの型が正しいか確認（テキスト vs セレクト）
