## 経過措置医薬品患者抽出チェックリスト（Phase 1-4）

**作成日**: 2026年3月11日 18:33
**期限**: 2026年3月31日
**優先成分**: リラグルチド、アモキサピン、トラニラスト

### Phase 1: MedicalCareStation患者抽出（ゆうすけ操作）

**所要時間**: 約30分
**担当**: ゆうすけ

- [ ] Step 1: MedicalCareStationを起動
- [ ] Step 2: 患者検索画面を開く
- [ ] Step 3: 処方薬検索タブを選択
- [ ] Step 4: 優先成分名を入力（リラグルチド）
- [ ] Step 5: 検索期間を設定（過去6ヶ月）
- [ ] Step 6: 検索実行
- [ ] Step 7: 結果をCSVエクスポート
- [ ] Step 8: 同様にアモキサピン、トラニラストで繰り返し
- [ ] Step 9: CSVファイルをクラウドまたは共有フォルダに保存
- [ ] Step 10: 保守AIへ完了通知（Telegram）

**出力**: 患者CSVファイル（成分別）

---

### Phase 2: 患者リスト整理

**所要時間**: 約20分
**担当**: 保守AI（自動化可能）

#### 患者リスト整形テンプレート

| No | 患者ID | 患者名 | 処方薬 | 用法用量 | 処方医 | 最終処方日 | 優先度 | 備考 |
|----|--------|--------|--------|----------|--------|------------|--------|------|
| 1 | | | | | | | 高/中/低 | |
| 2 | | | | | | | 高/中/低 | |

**優先度判定基準**:
- **高**: 他剤併用なし・代替困難・最終処方日が直近
- **中**: 代替薬あり・経過観察可能
- **低**: 他剤併用あり・代替容易

- [ ] Step 11: CSVファイルから患者リスト整形テンプレートに転記
- [ ] Step 12: 重複患者の統合（複数成分処方の場合）
- [ ] Step 13: 優先度判定（高/中/低）
- [ ] Step 14: 処方医別にグループ化
- [ ] Step 15: 整理済みリストを保存・trainerへ報告

**出力**: 整理済み患者リスト（Excel/Markdown）

---

### Phase 3: 処方医への相談

**所要時間**: 患者数による
**担当**: ゆうすけ

#### 処方医相談FAX雛形

```
【相談依頼書】

日付: 2026年3月XX日
宛先: XX医院 XX先生
発信: XX薬局 薬剤師 氏名

【件名】経過措置医薬品に伴う処方変更のご相談

【患者情報】
患者名: [患者名]
処方薬: [成分名] [用法用量]
処方期間: [開始日]〜[最終処方日]

【相談内容】
上記患者様が服用中の[成分名]は、2027年3月末をもって薬価削除となる
経過措置医薬品です。代替薬への変更をご検討いただきたく存じます。

【代替薬候補】
1. [代替薬1] [用法用量]
2. [代替薬2] [用法用量]

【ご回答期限】
2026年3月XX日までにご回答いただけますと幸いです。

【連絡先】
電話: XXX-XXXX-XXXX
FAX: XXX-XXXX-XXXX

以上、よろしくお願いいたします。
```

- [ ] Step 16: 処方医別に患者リストを送付準備
- [ ] Step 17: FAXまたは電子送信
- [ ] Step 18: 回答待ちリストを作成
- [ ] Step 19: 回答受領時に患者カードに記録

**出力**: 相談依頼書送付済みリスト

### Phase 4: 患者への説明

**所要時間**: 患者数による
**担当**: ゆうすけ

#### 患者説明チェックリスト

- [ ] Step 20: 代替薬決定患者のリスト作成
- [ ] Step 21: 次回来局時に説明実施
- [ ] Step 22: 患者同意書の署名取得（必要時）
- [ ] Step 23: 薬歴に変更理由を記録
- [ ] Step 24: 変更後の服薬状況をモニタリング

#### 患者説明文例

```
[患者名]様

現在服用中の[薬品名]についてご説明いたします。
このお薬は製造終了となり、来年3月以降は処方できなくなります。
そこで、[代替薬名]への変更をご案内しております。
効果は同程度ですが、念のため服用後の体調変化をお知らせください。
```

**出力**: 患者説明完了リスト
