## ◆ 2. transition-drugs-387-matcher.py 入力ファイル要件

### 必要なCSVファイル（2種類）

#### ① MCSからエクスポートする処方データ（`prescriptions_export.csv`）

| 必須カラム名 | 内容 | MCS操作での確認ポイント |
|------------|------|----------------------|
| `患者ID` | 患者識別番号 | MCS患者番号をそのまま使用 |
| `患者名` | 患者氏名 | |
| `薬品名` | 処方されている薬品名（先発・後発どちらも） | スクリプトはこのカラムで照合する |
| `規格` | mg等の規格 | あれば精度が上がる（なくても動作可） |
| `用法` | 1日○回等 | なくても動作可 |
| `処方日` | 処方年月日 | なくても動作可 |

> ⚠️ **最重要**: `薬品名` カラムが必須。MCSエクスポート時にカラム名が異なる場合は、スクリプト内の `drug_name_col='薬品名'` の部分を合わせて変更する必要あり（ゆうすけには不要、必要なら薬剤師エージェントが対応）。

**MCSエクスポート手順（最小ステップ）:**
1. MedicalCareStation を開く
2. 患者検索 → 「薬剤検索」タブ
3. 検索条件: 処方中（終了日なし）の全患者
4. 結果をCSVエクスポート → ファイル名 `prescriptions_export.csv` で保存
5. スクリプトと同じフォルダに置く（`~/Desktop/openclaw-cowork/homecare/`）

#### ② 387品目リスト（`drugs_387_list.csv`）

| カラム名 | 内容 |
|---------|------|
| `薬品コード` | 薬品コード（任意） |
| `薬品名` | 削除対象の薬品名 ← 照合に使用 |
| `規格` | 規格（任意） |
| `メーカー` | メーカー名（任意） |
| `削除タイプ` | 「経過措置」「成分単位削除」「即時削除」のいずれか |

> 📋 `drugs_54_component_delete.csv`（54品目のうち登録済み）は既に存在。387品目の完全版は `r8-transition-drugs-list-387.pdf` を参照し、必要な成分を追加する（全量追加は不要 — 在宅患者への影響が高いもの優先）。

## ◆ 4. 抽出後のA/B/C/D分類ワークフロー（簡潔版）

スクリプト実行後、対象患者を以下の4分類に振り分ける:

| 分類 | 定義 | 対応 | 期限 |
|------|------|------|------|
| **A** | 成分単位削除（代替薬必須） | 主治医に処方変更FAX + 患者説明 | **3月末〜4月中** |
| **B** | 経過措置中だが早期切り替え推奨（高齢・認知症・在宅継続中） | 次回訪問時に代替薬提案 | 2027年3月前 |
| **C** | 経過措置中・現状維持可 | 経過観察・定期フォロー | 2027年3月前 |
| **D** | 既に代替薬に切り替え済み or 今後処方なし | 記録のみ・対応不要 | 不要 |

**カテゴリAの詳細対応フロー:**
```
① 患者確認 → ② 代替薬を薬剤師エージェントで評価（既存シート参照）
→ ③ 主治医へFAX（テンプレート: teriparatide-action-kit-2026-03.md に記載）
→ ④ 患者・家族への説明 → ⑤ 新処方受領 → ⑥ 新薬お届け・服薬指導
```
