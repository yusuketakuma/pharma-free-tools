# 自動化ツール比較: n8n vs Make vs Zapier

## 目的
副業収益化の自動導線構築における最適ツール選定

## 比較サマリー

| 項目 | n8n | Make | Zapier |
|------|-----|------|--------|
| **月額費用** | 無料（セルフホスト）| $9〜 | $19.99〜 |
| **技術難易度** | 中（Docker知識必要）| 低 | 極低 |
| **AI連携** | ◎（OpenAI/Anthropic直接）| ○ | ○ |
| **日本語対応** | △ | ○ | ◎ |
| **制御性** | ◎（コード実行可能）| ○ | △ |
| **安定性** | ○（自管理必要）| ◎ | ◎ |

## 詳細分析

### n8n（推奨: 技術者向け）

**メリット**
- 完全無料（セルフホスト）
- AIエージェント連携に強い（OpenClawとの親和性高い）
- JavaScript/Python実行可能
- GitHubとの連携容易

**デメリット**
- セルフホストの運用負荷
- Docker知識必要
- 障害時は自己復旧

**月5万円事例の再現性**
- peaceflat事例: n8n + AI × SNS自動投稿で月5万円達成
- 技術的知識不要とされるが、初期設定は必要
- テンプレート活用で時短可能

**構築コスト**
- 初期: 2-4時間（Docker環境構築 + ワークフロー設計）
- 運用: 月1-2時間（ログ確認・障害対応）

### Make（推奨: 非技術者向け）

**メリット**
- コスト効率が高い（Zapierの1/3〜1/5）
- 分岐・ルーティングに強い
- 視覚的エディタが直感的

**デメリット**
- 無料枠が小さい（1000オペレーション/月）
- AI連携はn8nより制限あり

**適用ケース**
- 非技術者がメイン運用
- 定型業務自動化（メール・SNS投稿・Notion連携）
- コスト重視

### Zapier（推奨: 導入容易性重視）

**メリット**
- 導入が最も容易
- 連携アプリ数最多（5000+）
- 日本語ドキュメント充実

**デメリット**
- 高コスト（$50+/月で実用レベル）
- AI制御はn8nより劣る
- 複雑な分岐は苦手

**適用ケース**
- まずは試してみたい
- 技術リソースなし
- 予算あり

## 技術メモ

### n8nセルフホスト構築（Mac mini）

```bash
# Dockerインストール済み前提
docker run -it --rm \
  --name n8n \
  -p 5678:5678 \
  -v ~/.n8n:/home/node/.n8n \
  n8nio/n8n

# アクセス: http://localhost:5678
```

### 推奨ワークフローテンプレート

1. **Gumroad販売通知**
   - Trigger: Gumroad Webhook
   - Action: Gmail送信 + Notion記録

2. **SNS自動投稿**
   - Trigger: Notion新規レコード
   - Action: X API投稿

3. **メールマガジン自動化**
   - Trigger: Beehiiv新規購読者
   - Action: ウェルカムメール送信
